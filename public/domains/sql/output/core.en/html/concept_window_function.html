Window Function

Window Function

Hook. Imagine shazam-identifying a song from a 5-second clip. The app cannot analyze an infinite signal — it must isolate a short, well-behaved excerpt. The mathematical tool that does this cleanly is the window function.

Definition. A window function is a function $w : \mathbb{R} \to [0,1]$ with compact support — meaning $w(t) = 0$ outside some finite interval $[a, b]$. When you multiply a signal $f(t)$ by $w(t)$, the product $g(t) = f(t)\, w(t)$ retains only the portion of $f$ inside the window. Formally, if $\text{supp}(w) = [a,b]$, then

$$g(t) = f(t)\, w(t) = 0 \quad \text{for all } t \notin [a, b].$$

Worked example. The simplest window is the rectangular window on $[0, T]$:

$$w_{\text{rect}}(t) = \begin{cases} 1 & 0 \le t \le T \\ 0 & \text{otherwise.} \end{cases}$$

Applying it to a cosine $f(t) = \cos(2\pi \nu t)$ gives a finite-duration tone $g(t) = \cos(2\pi \nu t)\cdot w_{\text{rect}}(t)$, which is now integrable and amenable to Fourier analysis.

Why it matters. The Fourier transform $\hat{f}(\xi) = \int_{-\infty}^{\infty} f(t)\,e^{-2\pi i \xi t}\,dt$ requires $f \in L^1(\mathbb{R})$. A persistent sinusoid is not in $L^1$, but $g = f\cdot w$ is, because compact support guarantees $\int |g|\,dt \le \|f\|_\infty \cdot (b-a) < \infty$. Windowing is therefore the bridge from idealized infinite signals to computable, finite transforms. Different window shapes (Hann, Hamming, Blackman) trade off frequency resolution against spectral leakage, a subtle property studied via the window's own Fourier transform $\hat{w}$.

Practice. Let $w(t) = \max(0,\, 1 - |t|)$ (the triangular, or Bartlett, window). Identify its support, sketch its graph, and verify that $\int_{-\infty}^{\infty} w(t)\,dt = 1$.
