Index

When you look up a word in a dictionary, you don't read every page — you jump straight to the right letter. An index in mathematics serves the same purpose: it is a label that pinpoints a specific element inside a collection without ambiguity.

Definition. Given a sequence or collection of objects $a_1, a_2, a_3, \ldots, a_n$, the index is the subscript $i$ in $a_i$ that identifies which element you are referring to. More precisely, an index is an element of an index set $I$, and an indexed family is a function $f: I \to S$ where $S$ is some set of values. We write the image of $i$ as $a_i$ rather than $f(i)$ to emphasize the positional role of $i$.

Worked example. Consider the sequence of squares $a_i = i^2$ for $i \in \{1, 2, 3, 4, 5\}$:

$$a_1 = 1,\quad a_2 = 4,\quad a_3 = 9,\quad a_4 = 16,\quad a_5 = 25.$$

The index $i = 3$ selects exactly $a_3 = 9$. Summation notation packages all five values cleanly:

$$\sum_{i=1}^{5} a_i = \sum_{i=1}^{5} i^2 = 1 + 4 + 9 + 16 + 25 = 55.$$

Here the index $i$ is a bound variable — it exists only inside the sum and carries no meaning outside it.

Why it works. An index set imposes an ordering (or at least a labeling) on what would otherwise be an unstructured collection. Because each label maps to exactly one element, retrieval is unambiguous. This is precisely the definition of a function, so indexed families inherit all the machinery of function theory — domains, ranges, injectivity — for free.

Practice problem. Let $b_k = 2k - 1$ for $k \in \{1, 2, \ldots, n\}$.
