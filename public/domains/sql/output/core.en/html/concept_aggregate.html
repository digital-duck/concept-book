Aggregate

Aggregate

Every morning, a city traffic dashboard collapses millions of GPS pings into a single number: average commute time. That act of collapsing many values into one is aggregation — and it sits at the foundation of probability, statistics, and algorithm analysis.

Definition. Given a sequence of values $x_1, x_2, \ldots, x_n$ from some set $S$ and a binary operation $\oplus : S \times S \to S$ that is associative, the aggregate of the sequence under $\oplus$ is

$$\bigoplus_{i=1}^{n} x_i \;=\; x_1 \oplus x_2 \oplus \cdots \oplus x_n.$$

Common aggregates include the sum ($\oplus = +$), product ($\oplus = \times$), maximum, minimum, and logical conjunction. When $\oplus$ is also commutative and has an identity element $e$, the aggregate extends naturally to multisets and empty sequences (returning $e$).

Worked example. Let $x_1 = 3,\; x_2 = 7,\; x_3 = 2,\; x_4 = 5$. The sum aggregate is

$$\sum_{i=1}^{4} x_i = 3 + 7 + 2 + 5 = 17,$$

while the max aggregate is $\max(3, 7, 2, 5) = 7$. Notice they answer different questions: sum measures total, max identifies the peak.

Why associativity matters. Associativity guarantees that the order of parenthesization does not change the result:

$$(x_1 \oplus x_2) \oplus x_3 = x_1 \oplus (x_2 \oplus x_3).$$

This makes aggregates parallelizable — you can split the sequence into chunks, aggregate each chunk independently, then aggregate the chunk results. Without associativity, splitting the computation would yield different answers depending on the split boundary.

Practice problem. For $x_1 = 2,\; x_2 = 4,\; x_3 = 8,\; x_4 = 16$, compute:

  • $\prod_{i=1}^{4} x_i$
  • $\min\bigl(x_1, x_2, x_3, x_4\bigr)$
  • Verify that splitting the product as $\bigl(x_1 \cdot x_2\bigr)\cdot\bigl(x_3 \cdot x_4\bigr)$ gives the same result as the sequential computation.
