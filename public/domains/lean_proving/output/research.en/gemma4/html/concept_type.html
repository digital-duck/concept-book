Definition. A type system $\mathcal{T}$ formalizes a hierarchy of mathematical structures used to categorize terms and establish consistency across computation or logical deduction. Formally, $\mathcal{T}$ defines a collection of sorts, where each sort $A$ is itself viewed as an element of a meta-type (e.g., $\text{Type}_i$). A term $t$ inhabits a type $A$, denoted $\Gamma \vdash t: A$. The fundamental structural requirement for $\mathcal{T}$ is the formation rule, which dictates how types are constructed from base sorts and previously formed types. Standard formations include product $(\times)$, function space $(\to)$, and crucially, dependent sum ($\Sigma$). Given a type $B$ and a type family $P$ indexed by $B$, the formation of $\Sigma$-type, denoted $\Sigma x: B. P(x)$, constructs a type whose inhabitants are pairs $(b, p)$ such that $p$ has type $P(b)$.

For any simply typed lambda calculus augmented with base types and dependent pairing ($\Sigma$), two terms $t_1$ and $t_2$ have the same type if and only if they are convertible according to $\beta$-reduction and definitional equality within the assigned type structure. That is, if $\Gamma \vdash t_1: A$ and $\Gamma \vdash t_2: A$, then $t_1 \equiv_{\beta} t_2$.
