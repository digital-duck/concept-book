🍃 Inductive Types

Inductive types provide the fundamental mechanism in type theory for defining complex algebraic data structures. Formally, an inductive type $A$ is defined by specifying a set of constructors $\{C_i\}$ that dictate how elements are synthesized from arguments drawn from previously defined types (the sorts). If $A$ is defined by a collection of constructors $\vec{C}: T_{1} \rightarrow \dots \rightarrow T_{n} \rightarrow A$, then any element $a : A$ must be constructed via the application of these designated constructors to arguments. This structural constraint ensures that every value possesses a unique decomposition relative to its generating basis types, establishing a canonical representation for elements of $A$.

The central result concerning inductive definitions is the Principle of Structural Recursion (or Induction): For any type $A$ defined by $\{C_i\}$, and given an arbitrary set of mapping functions $\{f_i: \text{Domain}(C_i) \rightarrow B\}$ into a target type $B$, there exists a unique total function $r: A \rightarrow B$. This map $r$ is characterized by the property that for any element constructed as $a := C(x_1, \dots, x_n)$, its value must be computed by applying the corresponding function $f$ to the components: $r(a) = f(x_1, \dots, x_n)$.

The proof of this principle relies critically on the inherent well-foundedness induced by the construction method. Since every element $a : A$ is built from base types through a finite sequence of constructor applications, the structural dependency graph formed by the elements of $A$ must be a Directed Acyclic Graph (DAG). The unique existence and definition of $r$ are established via transfinite induction over the structure of $A$, ensuring that the function definition proceeds correctly for all possible paths without encountering circular dependencies. This guarantees both type safety and termination for any computation defined by such recursion.

Significantly, inductive types provide a constructive model corresponding precisely to initial algebra semantics. They admit canonical models where equality is decidable (i.e., $a = b$ if and only if their unique structural decomposition matches), which is crucial for formal verification in proof assistants. A major generalization of this concept involves co-inductive types, which define structures not through construction but via observation (or initial coalgebra semantics). These are typically applied to modeling potentially infinite processes or reactive systems that require techniques such as greatest fixed points rather than least fixed points.
