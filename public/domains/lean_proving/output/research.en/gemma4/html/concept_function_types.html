🍃 Function Types

A function type defines the structural relationship between the domain and codomain of a mapping. Formally, if $\tau_1$ and $\tau_2$ are types in a given type system, the function type $\tau_1 \to \tau_2$ represents the set (or class) of all total functions $f: D \to C$. These functions are fundamentally constructed via $\lambda$-abstraction. We define an abstraction term $\lambda x: \sigma. e$, where $x$ is a variable, $\sigma$ is its type, and $e$ is the body of type $\tau_2$. The typing judgment establishes that this abstraction yields a function term with type $\sigma \to \tau_2$: $\Gamma \vdash (\lambda x: \sigma. e): \sigma \to \tau_2$. This concept forms the basis for the semantics of lambda calculus ($\lambda$) and is generalized in dependent type theory (DTT).

Theorem (Type Compatibility for Function Application): Given a function $g$ with type $\sigma \to \tau$, and an argument $x$ with type $\sigma$, the application $g(x)$ is well-typed, and its resulting type is $\tau$. That is, if $\Gamma \vdash g: \sigma \to \tau$ and $\Gamma \vdash x: \sigma$, then $\Gamma \vdash g(x): \tau$.

Proof: The proof relies on the inductive definition of function application ($\text{App}$). By definition, a term $g$ having type $\sigma \to \tau$ must be an abstraction over an input of type $\sigma$. The typing judgment for application is defined precisely to enforce this compatibility:

$$ \frac{\Gamma \vdash g: \tau_1 \to \tau_2 \quad \Gamma \vdash x: \tau_1}{\Gamma \vdash g(x): \tau_2} $$
