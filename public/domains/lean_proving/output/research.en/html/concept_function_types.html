Function Types

Definition. Let $A$ and $B$ be types in a simply-typed system $\lambda^\to$. The function type $A \to B$ (also written $B^A$ in categorical notation) is the type whose terms are total functions from $A$ to $B$. Formally, if $\Gamma, x : A \vdash e : B$ is derivable, then $\Gamma \vdash \lambda x.\, e : A \to B$ by the abstraction rule ($\lambda$-introduction). Conversely, the application rule ($\lambda$-elimination) states: if $\Gamma \vdash f : A \to B$ and $\Gamma \vdash a : A$, then $\Gamma \vdash f\, a : B$.

Theorem (Curry–Howard correspondence). In the simply-typed lambda calculus, the type $A \to B$ corresponds bijectively to the proposition $A \Rightarrow B$ under the Curry–Howard isomorphism: terms of type $A \to B$ are in natural bijection with proofs of the implication $A \Rightarrow B$ in intuitionistic propositional logic.

Proof. One constructs a functor $\Phi$ from the category of typing derivations to the category of intuitionistic proof trees. The abstraction rule maps exactly to the $\Rightarrow$-introduction rule, and application maps to $\Rightarrow$-elimination (modus ponens). $\beta$-reduction corresponds to proof normalisation; $\eta$-expansion corresponds to proof extensionality. The bijection is canonical and respects composition, establishing a category equivalence. $\square$

Remark. In categorical semantics, function types are precisely the exponential objects $B^A$ in a Cartesian closed category (CCC): an object equipped with a natural bijection $\mathrm{Hom}(C \times A,\, B) \cong \mathrm{Hom}(C,\, B^A)$ for all objects $C$. The existence of all such exponentials characterises CCCs (Lambek–Scott, Introduction to Higher-Order Categorical Logic, 1986), making this the canonical denotational model for $\lambda^\to$. The $A \to B$ construction is then the non-dependent base case of a broader hierarchy—generalising to dependent and resource-sensitive variants in richer type theories—but each such extension introduces its own primitive and is treated separately.
