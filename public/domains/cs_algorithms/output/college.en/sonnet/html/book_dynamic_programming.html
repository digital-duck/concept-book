🌱 Memory Access

Definition. Memory access refers to the operation by which a program reads from or writes to a location in a computer's memory hierarchy. Each access is characterized by an address $a \in [0, M)$, where $M$ is the total addressable memory size, a type (read or write), and a latency $\ell$ that depends on which level of the hierarchy services the request. The hierarchy is commonly modeled as a sequence of caches $C_1, C_2, \ldots, C_k$ followed by main memory, with capacities $S_1 < S_2 < \cdots < S_k < M$ and latencies $\ell_1 < \ell_2 < \cdots < \ell_k < \ell_{\text{mem}}$.

The External Memory Model (Aggarwal–Vitter, 1988) formalizes this as a two-level hierarchy: a fast internal memory of size $M$ blocks and a slow external disk. An algorithm's cost is measured in block transfers (I/Os) rather than individual word accesses, since transferring a block of $B$ words costs the same as transferring one word at the disk level.

Worked Example. Consider scanning an array $A$ of $N$ integers stored contiguously. A sequential scan reads each element exactly once, loading $\lceil N/B \rceil$ blocks. Thus the I/O complexity is $\Theta(N/B)$. In contrast, if the same $N$ elements were accessed in a uniformly random order (as in a naive hash-table probe), the expected number of cache misses is $\Theta(N)$ because each access likely hits a distinct block, even when $N \operatorname{gg} M$. This gap — $\Theta(N/B)$ vs.\ $\Theta(N)$ — illustrates the penalty for poor spatial locality.

Key Theorem (Scanning Lower Bound). Any algorithm that reads $N$ elements stored on disk must perform at least $\Omega(N/B)$ block transfers, regardless of the access pattern, because each transfer can deliver at most $B$ new elements. For algorithms exhibiting temporal locality (reuse within a working set of size $\le M$), the effective I/O cost can be reduced to $O(N/B)$, matching the scanning lower bound.

🌱 Recursion

🌱 Recursion

Definition. A function $f$ is recursive if its definition references itself. Formally, a recursive algorithm solves an instance of size $n$ by reducing it to one or more instances of strictly smaller size, together with a base case that is solved directly without further recursion. Every correct recursive algorithm must satisfy two properties: (1) the base case is reachable and solved in finite time, and (2) each recursive call strictly decreases some well-founded measure (typically the input size), guaranteeing termination.

Worked Example: Factorial. Define $n! = n \cdot (n-1)!$ for $n \geq 1$, with $0! = 1$. The corresponding algorithm is:

$$ \texttt{fact}(n) = \begin{cases} 1 & \text{if } n = 0 \\ n \cdot \texttt{fact}(n-1) & \text{if } n \geq 1 \end{cases} $$

Tracing $\texttt{fact}(4)$:

$$ \texttt{fact}(4) \to 4 \cdot \texttt{fact}(3) \to 4 \cdot 3 \cdot \texttt{fact}(2) \to 4 \cdot 3 \cdot 2 \cdot \texttt{fact}(1) \to 4 \cdot 3 \cdot 2 \cdot 1 \cdot \texttt{fact}(0) = 24. $$

The call stack grows to depth $n$ before unwinding; each frame consumes $O(1)$ space, so the total space complexity is $\Theta(n)$. The time complexity satisfies the recurrence $T(n) = T(n-1) + O(1)$, which resolves to $T(n) = \Theta(n)$.

Key Theorem (Master Theorem — Special Case). For a divide-and-conquer recurrence of the form $T(n) = a \, T\!\left(\tfrac{n}{b}\right) + \Theta(n^d)$, where $a \geq 1$, $b > 1$, and $d \geq 0$:

$$ T(n) = \begin{cases} \Theta(n^d \log n) & \text{if } a = b^d \\ \Theta(n^{\log_b a}) & \text{if } a > b^d \\ \Theta(n^d) & \text{if } a < b^d \end{cases} $$

This theorem provides a mechanical method for analyzing the complexity of recursive algorithms without manually unrolling recurrences.

🍃 Dynamic Programming

🍃 Dynamic Programming

Definition. Dynamic programming (DP) is an algorithm design paradigm that solves complex optimization or counting problems by decomposing them into overlapping subproblems, solving each subproblem exactly once, and storing results for reuse. A problem admits a DP solution when it exhibits two structural properties: optimal substructure (an optimal solution to the whole problem contains optimal solutions to its subproblems) and overlapping subproblems (the same subproblems recur across different branches of a naive recursive computation).

Formally, let $\text{OPT}(i)$ denote the value of an optimal solution restricted to the first $i$ elements of the input. A DP formulation expresses $\text{OPT}(i)$ as a recurrence over a constant number of smaller indices, together with a base case $\text{OPT}(0)$.

Worked Example: Weighted Interval Scheduling. Given $n$ jobs with start times $s_j$, finish times $f_j$, and weights $w_j$, find a conflict-free subset of maximum total weight. Sort jobs by finish time. Define $p(j)$ as the largest index $i < j$ such that job $i$ is compatible with job $j$. The recurrence is

$$ \text{OPT}(j) = \max\bigl(w_j + \text{OPT}(p(j)),\; \text{OPT}(j-1)\bigr), \quad \text{OPT}(0) = 0. $$

The two choices are: include job $j$ (gaining $w_j$ and skipping to $p(j)$) or exclude it (retaining $\text{OPT}(j-1)$). Filling the table left-to-right in $O(n)$ time (after an $O(n \log n)$ preprocessing sort and binary search for each $p(j)$) yields the overall $O(n \log n)$ algorithm.

Key Theorem (Bellman's Principle of Optimality, 1957). A policy is globally optimal if and only if, for every intermediate state reached under that policy, the remaining decisions form an optimal policy for the subproblem starting from that state. This theorem is the formal justification that replacing a suboptimal sub-solution always produces a globally better or equal solution, making the DP recurrence correct.

Payoff 🎯

Definition. Dynamic programming (DP) is an algorithm design paradigm that solves an optimization or counting problem by decomposing it into overlapping subproblems, solving each exactly once, and storing results in a table so that each stored value is reused in $O(1)$ time. Formally, a problem admits a DP solution when it satisfies two structural properties: optimal substructure — an optimal solution to the whole incorporates optimal solutions to its subproblems — and overlapping subproblems — the same sub-instances recur across different branches of a naive recursion. Together these properties transform an exponential search space into a polynomial one.

Worked Example. Consider the classic 0/1 knapsack problem: given $n$ items with weights $w_i$ and values $v_i$ and a capacity $W$, maximize $\sum v_i x_i$ subject to $\sum w_i x_i \le W$, $x_i \in \{0,1\}$. Define $\text{dp}[i][c]$ as the maximum value attainable using only the first $i$ items with remaining capacity $c$. The recurrence is

$$ \text{dp}[i][c] = \max\!\bigl(\text{dp}[i-1][c],\; v_i + \text{dp}[i-1][c - w_i]\bigr), \quad c \ge w_i, $$

with $\text{dp}[0][c] = 0$ for all $c$. Filling the $n \times W$ table row by row yields a time complexity of $\Theta(nW)$, converting a brute-force $O(2^n)$ enumeration into a pseudo-polynomial algorithm. Each cell is computed once; the exponential blow-up is entirely absorbed by the table.

Key Theorem (Bellman's Principle of Optimality, 1957). An optimal policy has the property that, whatever the initial state and decision, the remaining decisions must constitute an optimal policy with regard to the state resulting from the first decision. This principle is the formal guarantee that local subproblem solutions compose into a globally optimal answer, and it underlies every DP correctness argument.

Connection to Applications. The payoff of mastering DP extends immediately across disciplines. In search engine ranking, PageRank's iterative value propagation mirrors DP on a graph of pages. In compiler optimization, register allocation and instruction scheduling reduce to shortest-path DP over control-flow graphs. In genomic sequence alignment, the Smith–Waterman algorithm fills a scoring matrix whose recurrence is structurally identical to the knapsack table above, enabling alignment of DNA reads against reference genomes in $O(mn)$ time. In compression, the Lempel–Ziv family and Huffman-based schemes rely on DP to construct optimal prefix codes and maximal dictionary entries.
