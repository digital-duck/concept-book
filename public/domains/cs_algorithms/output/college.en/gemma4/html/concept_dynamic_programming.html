🍃 Dynamic Programming

Definition Dynamic Programming (DP) is a powerful algorithmic technique used primarily for solving complex problems that exhibit optimal substructure and overlapping subproblems. It operates on the principle of breaking down a large problem into smaller, interconnected subproblems. Crucially, instead of re-solving the same subproblem multiple times (as might happen in naive recursion), DP stores the results of these subproblems—often in a table or memoization structure—and refers back to these stored values when they are needed again. This approach transforms an exponential time complexity algorithm into a polynomial one. Mathematically, if $P$ is the original problem and $S_1, S_2, \dots, S_k$ are its subproblems, DP ensures that the solution for any subproblem $S_i$ is computed exactly once. This efficiency gain is fundamental to optimization problems in graph theory, combinatorial mathematics, and sequence alignment.

Worked Example: Fibonacci Sequence Consider calculating the $n$-th Fibonacci number, $F(n)$, defined by the recurrence relation $F(n) = F(n-1) + F(n-2)$, with base cases $F(0)=0$ and $F(1)=1$. A purely recursive approach calculates $F(5)$ by recursively calculating $F(4)$ and $F(3)$. Notice that $F(3)$ will be calculated multiple times (once via $F(4)$ and once via the direct call to $F(3)$). Using DP, we build a memoization table (or bottom-up array) where we compute $F(2), F(3), \dots, F(n)$ sequentially. We define an array $DP[i]$ to store $F(i)$. The recurrence is then implemented iteratively:

$$ DP[i] = DP[i-1] + DP[i-2], \quad \text{for } i \ge 2. $$

This iterative approach reduces the time complexity from $O(\phi^n)$ (exponential) to $O(n)$ (linear).

Key Theorem: Optimal Substructure Property A problem possesses optimal substructure if an optimal solution to the overall problem can be constructed from optimal solutions to its subproblems. This property is necessary, but not sufficient, for DP. For example, in the shortest path problem on a Directed Acyclic Graph (DAG), finding the shortest path from $u$ to $v$ requires knowing that the subpath from $u$ to any intermediate node $w$ must itself be the shortest possible path between those two points. This principle allows us to use Bellman-Ford or Dijkstra's algorithm, which are classic DP applications, because the overall optimal solution is built piece by local optimal decisions.
