🍃 Binary Tree

A binary tree is an ordered generalization of a tree data structure in which each node has at most two children, typically referred to as the left child and the right child. Formally, if $T$ is a set of nodes, $r \in T$ is designated as the root. For any non-root node $v$, its parent $\text{parent}(v)$ must exist in $T$. The structure imposes strict constraints: every node $v$ can have at most one left child and at most one right child. This constraint fundamentally limits the branching factor to two, making binary trees highly efficient for representing hierarchical data such as file systems or expression parsings. We denote a tree $T$ by its set of nodes $V(T)$ and its edge set $E(T)$.

Consider the process of building a Binary Search Tree (BST) from an input sequence of keys $\{k_1, k_2, \dots, k_N\}$. The first key becomes the root. For any subsequent node $v$, all keys in its left subtree must be less than $v$'s key, and all keys in its right subtree must be greater than $v$'s key. This ordering property is critical for efficient search operations.
