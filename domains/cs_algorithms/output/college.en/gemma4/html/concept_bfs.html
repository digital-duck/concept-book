🍃 Bfs

🍃 BFS

Breadth-First Search (BFS) is a graph traversal algorithm used to explore all nodes in an arbitrary graph level by level. Given a connected graph $G = (V, E)$ and a designated starting node $s \in V$, BFS systematically explores the neighborhood of $s$ before moving on to the neighbors of those neighbors, and so forth. This process guarantees that nodes are visited in non-decreasing order of their distance from $s$. The algorithm typically utilizes a queue data structure (FIFO: First-In, First-Out) to manage the set of nodes to be processed, ensuring that all nodes at depth $k$ are fully explored before any node at depth $k+1$ is considered. To prevent infinite loops in graphs containing cycles and to ensure each node is processed only once, BFS maintains a set or array tracking visited nodes.

Consider a concrete graph $G=(V, E)$ where $V = \{A, B, C, D\}$ and the edges are $E = \{(A, B), (A, C), (B, D)\}$. If we initiate BFS starting at node $A$, the process proceeds as follows:

  1. Initialize Queue $Q$: $[A]$. Mark $A$ as visited.
  2. Dequeue $A$. Examine neighbors $(B, C)$. Enqueue unvisited neighbors: $Q = [B, C]$. Mark $B$ and $C$ as visited.
  3. Dequeue $B$. Examine neighbor $D$. Enqueue unvisited neighbor: $Q = [C, D]$. Mark $D$ as visited.
  4. Dequeue $C$. Examine neighbors (none unvisited). $Q = [D]$.
  5. Dequeue $D$. Examine neighbors (none unvisited). $Q = []$. The traversal completes. BFS successfully explored the graph in order: $A \to B, C \to D$.

Key Theorem: If a graph is represented by an adjacency list and BFS is initiated at node $s$, then the distance computed from $s$ to any reachable node $v$ using Breadth-First Search is guaranteed to be the shortest path length in terms of number of edges (i.e., it finds the minimum hop count). This property makes BFS ideal for solving unweighted shortest path problems, such as finding the closest facility or determining connectivity reachability.

Lab Exercise: Given a directed graph $G$ and two nodes $s$ and $t$, implement the BFS algorithm using a queue. Modify your implementation to not only determine if $t$ is reachable from $s$, but also to reconstruct one shortest path (in terms of edges) from $s$ to $t$. You must use a parent array or dictionary structure during traversal to record the predecessor node for each newly visited node, which allows for path backtracking.
