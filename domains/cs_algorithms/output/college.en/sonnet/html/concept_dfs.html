🍃 Dfs

🍃 Dfs

Definition. Let $G = (V, E)$ be a directed or undirected graph with $|V| = n$ vertices and $|E| = m$ edges. Depth-First Search (DFS) is a graph traversal algorithm that explores as far as possible along each branch before backtracking. Formally, DFS maintains a coloring of vertices: each vertex $v \in V$ is initially white (undiscovered), marked gray upon first visit, and colored black when all its neighbors have been fully explored. The algorithm records two timestamps for each vertex: $d[v]$ (discovery time) and $f[v]$ (finish time), satisfying $d[v] < f[v]$. The overall time complexity is $\Theta(n + m)$.

Worked Example. Consider the directed graph $G$ with $V = \{1, 2, 3, 4\}$ and edges $E = \{(1,2),(1,3),(2,4),(3,4)\}$. Running DFS from vertex $1$ (visiting neighbors in numerical order) proceeds as follows:

  1. Visit $1$: $d[1] = 1$; recurse on neighbor $2$.
  2. Visit $2$: $d[2] = 2$; recurse on neighbor $4$.
  3. Visit $4$: $d[4] = 3$, no unvisited neighbors; $f[4] = 4$.
  4. Backtrack to $2$: $f[2] = 5$.
  5. Backtrack to $1$, recurse on neighbor $3$: $d[3] = 6$.
  6. Neighbor $4$ already black; $f[3] = 7$, then $f[1] = 8$.

The resulting finish-time ordering $\langle 4, 2, 3, 1 \rangle$ (decreasing $f$) gives a valid topological sort of $G$.

Key Theorem (Parenthesis Theorem). For any two vertices $u, v \in V$ in a DFS of graph $G$, exactly one of the following holds:

$$ [d[u],\, f[u]] \cap [d[v],\, f[v]] = \emptyset, \quad [d[u],\, f[u]] \subset [d[v],\, f[v]], \quad \text{or} \quad [d[v],\, f[v]] \subset [d[u],\, f[u]]. $$

Proof sketch. Because DFS recurses on $v$ while $u$ is gray if and only if $v$ is a descendant of $u$ in the DFS forest, the discovery–finish intervals nest perfectly, never partially overlapping. This structural property underpins cycle detection, strongly connected component decomposition, and topological sorting.

Lab Exercise. Implement DFS on an adjacency-list representation of a directed graph. For each vertex, record $d[v]$ and $f[v]$. Use your implementation to (a) detect whether the graph contains a back edge (indicating a cycle), and (b) output vertices in reverse finish-time order and verify this constitutes a valid topological sort when no cycle exists.
