🌱 Memory Access

🌱 Memory Access

Definition. Memory access refers to the operation by which a program reads from or writes to a location in a computer's memory hierarchy. Each access is characterized by an address $a \in [0, M)$, where $M$ is the total addressable memory size, a type (read or write), and a latency $\ell$ that depends on which level of the hierarchy services the request. The hierarchy is commonly modeled as a sequence of caches $C_1, C_2, \ldots, C_k$ followed by main memory, with capacities $S_1 < S_2 < \cdots < S_k < M$ and latencies $\ell_1 < \ell_2 < \cdots < \ell_k < \ell_{\text{mem}}$.

The External Memory Model (Aggarwal–Vitter, 1988) formalizes this as a two-level hierarchy: a fast internal memory of size $M$ blocks and a slow external disk. An algorithm's cost is measured in block transfers (I/Os) rather than individual word accesses, since transferring a block of $B$ words costs the same as transferring one word at the disk level.

Worked Example. Consider scanning an array $A$ of $N$ integers stored contiguously. A sequential scan reads each element exactly once, loading $\lceil N/B \rceil$ blocks. Thus the I/O complexity is $\Theta(N/B)$. In contrast, if the same $N$ elements were accessed in a uniformly random order (as in a naive hash-table probe), the expected number of cache misses is $\Theta(N)$ because each access likely hits a distinct block, even when $N \operatorname{gg} M$. This gap — $\Theta(N/B)$ vs.\ $\Theta(N)$ — illustrates the penalty for poor spatial locality.

Key Theorem (Scanning Lower Bound). Any algorithm that reads $N$ elements stored on disk must perform at least $\Omega(N/B)$ block transfers, regardless of the access pattern, because each transfer can deliver at most $B$ new elements. For algorithms exhibiting temporal locality (reuse within a working set of size $\le M$), the effective I/O cost can be reduced to $O(N/B)$, matching the scanning lower bound.

🌱 Recursion

🌱 Recursion

Definition. A function $f$ is recursive if its definition references itself. Formally, a recursive algorithm solves an instance of size $n$ by reducing it to one or more instances of strictly smaller size, together with a base case that is solved directly without further recursion. Every correct recursive algorithm must satisfy two properties: (1) the base case is reachable and solved in finite time, and (2) each recursive call strictly decreases some well-founded measure (typically the input size), guaranteeing termination.

Worked Example: Factorial. Define $n! = n \cdot (n-1)!$ for $n \geq 1$, with $0! = 1$. The corresponding algorithm is:

$$ \texttt{fact}(n) = \begin{cases} 1 & \text{if } n = 0 \\ n \cdot \texttt{fact}(n-1) & \text{if } n \geq 1 \end{cases} $$

Tracing $\texttt{fact}(4)$:

$$ \texttt{fact}(4) \to 4 \cdot \texttt{fact}(3) \to 4 \cdot 3 \cdot \texttt{fact}(2) \to 4 \cdot 3 \cdot 2 \cdot \texttt{fact}(1) \to 4 \cdot 3 \cdot 2 \cdot 1 \cdot \texttt{fact}(0) = 24. $$

The call stack grows to depth $n$ before unwinding; each frame consumes $O(1)$ space, so the total space complexity is $\Theta(n)$. The time complexity satisfies the recurrence $T(n) = T(n-1) + O(1)$, which resolves to $T(n) = \Theta(n)$.

Key Theorem (Master Theorem — Special Case). For a divide-and-conquer recurrence of the form $T(n) = a \, T\!\left(\tfrac{n}{b}\right) + \Theta(n^d)$, where $a \geq 1$, $b > 1$, and $d \geq 0$:

$$ T(n) = \begin{cases} \Theta(n^d \log n) & \text{if } a = b^d \\ \Theta(n^{\log_b a}) & \text{if } a > b^d \\ \Theta(n^d) & \text{if } a < b^d \end{cases} $$

This theorem provides a mechanical method for analyzing the complexity of recursive algorithms without manually unrolling recurrences.

🍃 Dfs

🍃 Dfs

Definition. Let $G = (V, E)$ be a directed or undirected graph with $|V| = n$ vertices and $|E| = m$ edges. Depth-First Search (DFS) is a graph traversal algorithm that explores as far as possible along each branch before backtracking. Formally, DFS maintains a coloring of vertices: each vertex $v \in V$ is initially white (undiscovered), marked gray upon first visit, and colored black when all its neighbors have been fully explored. The algorithm records two timestamps for each vertex: $d[v]$ (discovery time) and $f[v]$ (finish time), satisfying $d[v] < f[v]$. The overall time complexity is $\Theta(n + m)$.

Worked Example. Consider the directed graph $G$ with $V = \{1, 2, 3, 4\}$ and edges $E = \{(1,2),(1,3),(2,4),(3,4)\}$. Running DFS from vertex $1$ (visiting neighbors in numerical order) proceeds as follows:

  1. Visit $1$: $d[1] = 1$; recurse on neighbor $2$.
  2. Visit $2$: $d[2] = 2$; recurse on neighbor $4$.
  3. Visit $4$: $d[4] = 3$, no unvisited neighbors; $f[4] = 4$.
  4. Backtrack to $2$: $f[2] = 5$.
  5. Backtrack to $1$, recurse on neighbor $3$: $d[3] = 6$.
  6. Neighbor $4$ already black; $f[3] = 7$, then $f[1] = 8$.

The resulting finish-time ordering $\langle 4, 2, 3, 1 \rangle$ (decreasing $f$) gives a valid topological sort of $G$.

Key Theorem (Parenthesis Theorem). For any two vertices $u, v \in V$ in a DFS of graph $G$, exactly one of the following holds:

$$ [d[u],\, f[u]] \cap [d[v],\, f[v]] = \emptyset, \quad [d[u],\, f[u]] \subset [d[v],\, f[v]], \quad \text{or} \quad [d[v],\, f[v]] \subset [d[u],\, f[u]]. $$

Proof sketch. Because DFS recurses on $v$ while $u$ is gray if and only if $v$ is a descendant of $u$ in the DFS forest, the discovery–finish intervals nest perfectly, never partially overlapping. This structural property underpins cycle detection, strongly connected component decomposition, and topological sorting.

🍃 Dynamic Programming

🍃 Dynamic Programming

Definition. Dynamic programming (DP) is an algorithm design paradigm that solves complex optimization or counting problems by decomposing them into overlapping subproblems, solving each subproblem exactly once, and storing results for reuse. A problem admits a DP solution when it exhibits two structural properties: optimal substructure (an optimal solution to the whole problem contains optimal solutions to its subproblems) and overlapping subproblems (the same subproblems recur across different branches of a naive recursive computation).

Formally, let $\text{OPT}(i)$ denote the value of an optimal solution restricted to the first $i$ elements of the input. A DP formulation expresses $\text{OPT}(i)$ as a recurrence over a constant number of smaller indices, together with a base case $\text{OPT}(0)$.

Worked Example: Weighted Interval Scheduling. Given $n$ jobs with start times $s_j$, finish times $f_j$, and weights $w_j$, find a conflict-free subset of maximum total weight. Sort jobs by finish time. Define $p(j)$ as the largest index $i < j$ such that job $i$ is compatible with job $j$. The recurrence is

$$ \text{OPT}(j) = \max\bigl(w_j + \text{OPT}(p(j)),\; \text{OPT}(j-1)\bigr), \quad \text{OPT}(0) = 0. $$

The two choices are: include job $j$ (gaining $w_j$ and skipping to $p(j)$) or exclude it (retaining $\text{OPT}(j-1)$). Filling the table left-to-right in $O(n)$ time (after an $O(n \log n)$ preprocessing sort and binary search for each $p(j)$) yields the overall $O(n \log n)$ algorithm.

Key Theorem (Bellman's Principle of Optimality, 1957). A policy is globally optimal if and only if, for every intermediate state reached under that policy, the remaining decisions form an optimal policy for the subproblem starting from that state. This theorem is the formal justification that replacing a suboptimal sub-solution always produces a globally better or equal solution, making the DP recurrence correct.

🌸 Compiler Optimization

🌸 Compiler Optimization

Definition. Compiler optimization is the systematic transformation of a program's intermediate representation (IR) to produce equivalent code that executes more efficiently—reducing runtime, memory usage, or code size—while preserving observable semantics. Formally, let $P$ denote a program and $\operatorname{llbracket} P \operatorname{rrbracket}$ its denotational semantics. An optimization $\mathcal{O}$ is sound if $\operatorname{llbracket} \mathcal{O}(P) \operatorname{rrbracket} = \operatorname{llbracket} P \operatorname{rrbracket}$ for all inputs. Optimizations are typically organized into passes applied to a control-flow graph (CFG) $G = (V, E)$, where each node $v \in V$ is a basic block and edges represent possible control transfers.

Worked Example: Constant Folding and Dead Code Elimination. Consider the following three-address IR:

$$ \begin{aligned} t_1 &\leftarrow 3 \times 4 \\ t_2 &\leftarrow t_1 + x \\ t_3 &\leftarrow 0 \times y \\ t_4 &\leftarrow t_2 + t_3 \end{aligned} $$

Constant folding evaluates compile-time constants: $t_1 \leftarrow 12$ and $t_3 \leftarrow 0$. Substituting yields $t_4 \leftarrow 12 + x + 0$. Algebraic simplification then reduces this to $t_4 \leftarrow 12 + x$, and $t_3$ becomes unreferenced. Dead code elimination removes any assignment whose result is never subsequently used; here $t_3 \leftarrow 0 \times y$ is removed entirely, eliminating the multiplication. The optimized IR computes the same value in two instructions rather than four.

Key Theorem (Dataflow Convergence). For a monotone dataflow framework $(L, \sqsubseteq, \mathcal{F})$—where $L$ is a finite lattice, $\sqsubseteq$ is the partial order, and $\mathcal{F}$ is a set of monotone transfer functions—the iterative dataflow algorithm converges to the greatest fixed point in at most $|V| \cdot \text{height}(L)$ iterations. This guarantees that analyses such as reaching definitions, live-variable analysis, and available expressions always terminate and yield the most precise solution the framework can express.

Payoff 🎯

Definition. A compiler optimization is a semantics-preserving program transformation $T : P \to P'$ such that $P'$ is superior to $P$ under some cost metric $\mathcal{C}$ — most commonly execution time, memory footprint, or instruction count — while maintaining the invariant $\operatorname{llbracket} P \operatorname{rrbracket} = \operatorname{llbracket} P' \operatorname{rrbracket}$, i.e., both programs compute identical input-output functions. Formally, the compiler constructs a sequence of passes $T = T_k \circ T_{k-1} \circ \cdots \circ T_1$ operating over an intermediate representation (IR), often a control-flow graph $G = (V, E)$ where each node $v \in V$ is a basic block and each directed edge $(u, v) \in E$ represents a possible transfer of control.

Worked Example. Consider the expression $e = 3 \times x + 3 \times y$. A constant-folding pass recognizes that the subexpression $3 \times (x + y)$ is algebraically equivalent and reduces the multiplication count from two to one, yielding the simplified IR node $\texttt{MUL}(3,\, \texttt{ADD}(x, y))$. A subsequent common subexpression elimination (CSE) pass scans the data-flow graph for repeated computations: if $x + y$ already resides in a temporary register $t_1$, the pass rewrites every later occurrence of $\texttt{ADD}(x, y)$ to a register read in $O(1)$ rather than recomputing in $O(n)$ over a loop body.

Key Theorem (Kildall's Fixed-Point Theorem). Let $\mathcal{L}$ be a complete lattice and $f : \mathcal{L} \to \mathcal{L}$ a monotone transfer function over a finite control-flow graph. Then the data-flow equations $\mathbf{x} = f(\mathbf{x})$ possess a greatest fixed point, reachable by iterating $f$ from $\top$ in at most $|V| \cdot h(\mathcal{L})$ steps, where $h(\mathcal{L})$ is the height of the lattice. This theorem is the theoretical anchor for correctness proofs of analyses such as liveness, reaching definitions, and available expressions — every classical optimization pass rests upon it.
